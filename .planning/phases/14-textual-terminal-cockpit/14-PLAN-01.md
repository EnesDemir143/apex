<objective>
Add an app-first Hermes-inspired Textual cockpit launched by `apex`, with classic Rich CLI kept only as a secondary fallback/dev command.
</objective>

<branching>
Before implementation, create/switch to feature branch: `feat/textual-terminal-cockpit`. Use standard feature branch naming (`feat/<slug>`), not `bet5/` or `codex/`. Keep commits for this phase on that branch.
</branching>

<locked_user_decision>
Use the app-first Hermes-style approach:
- `apex` opens the modern Textual cockpit by default.
- `apex tui` / `apex --tui` may exist as explicit aliases if simple.
- `apex analyze AAPL` remains available only as secondary classic/dev mode.
- TUI should include ticker/company selector, market/chart panels, and command-composer/slash-command behavior.
- Analysis prompt composer must allow optional global instructions and per-agent instructions.
</locked_user_decision>
