# Phase 14.1: TUI Market Panel + Terminal Chart — Context

**Gathered:** 2026-05-06  
**Status:** Inserted between Phase 14 and Phase 15

<domain>
## Phase Boundary

Wire the existing TUI market area to real/local OHLCV data and add an in-terminal chart view before report/history work begins. This keeps Phase 15 focused on reports/history/cache while making the cockpit visibly useful for selected tickers.
</domain>

<decisions>
## Implementation Decisions

- Keep this dependency-light: prefer Textual/Rich tables, Unicode sparklines, and compact OHLCV/indicator summaries over browser-grade charts.
- Do not revive Streamlit or add a heavy frontend stack for this phase.
- Reuse existing local market data seams (`MarketDataFetcher`, deterministic fallback, indicator functions) rather than adding a new data provider.
- `/chart` should open a dedicated chart screen and support ticker selection via the same slash/ticker picker pattern used by `/select`.
- Chart UX can be terminal-native, not Streamlit-identical: enough to inspect trend, volume, and basic indicators from the TUI.
</decisions>
